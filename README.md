# Controle e Conferência de Dados (CRUD Logistic) 📦

## 🎯 O Problema
Em operações logísticas, a falta de integridade de dados durante o cadastro e a conferência de itens gera gargalos operacionais, erros de inventário e prejuízos financeiros por falta de rastreabilidade.

## 🚀 A Solução
Um sistema de CRUD (Create, Read, Update, Delete) simplificado, focado em **consistência de dados** e **velocidade operacional**. Desenvolvido para servir como base para ferramentas de conferência de carga, cadastro de clientes e controle de fluxo.

## 🛠️ Tecnologias
- **Python**: Lógica de processamento e validação.
- **SQLite**: Persistência de dados leve e eficiente para operações locais/offline.
- **Pytest**: Garantia de que as regras de negócio e validações de dados funcionem como esperado.

## 🏗️ Diferenciais Técnicos
- Separação clara entre a Camada de Dados (Backend) e a Camada de Interface (Aplicação).
- Validações de integridade para evitar cadastros duplicados ou inconsistentes.
